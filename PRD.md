# OpenCode Canvas - 产品需求文档 (PRD)

## 1. 产品概述
OpenCode Canvas 是一款专为 **OpenCode (开源代码/代码片段)** 管理而设计的可视化 AI 编排工具。它将传统的代码管理与无限画布结合，允许开发者通过 AI 智能体对代码进行审查、重构、文档生成及模块化组织。

## 2. 核心价值
- **代码空间化管理**：将零散的代码片段 (OpenCode) 以节点形式分布在画布上，直观展示模块间的逻辑联系。
- **AI 辅助代码生命周期**：利用专门的 AI Agent（如架构师、测试员、文档员）对 OpenCode 进行全方位维护。
- **结构化知识图谱**：通过连线建立代码间的依赖、调用或参考关系，形成可视化的技术知识库。

## 3. 功能需求

### 3.1 无限画布 (Infinite Canvas)
- **平移与缩放**：支持鼠标滚轮缩放，按住空格或中键平移。
- **网格背景**：提供视觉参考，支持对齐。
- **右键菜单**：在画布空白处右键可快速创建：
    - 主智能体 (Main Agent)
    - 工具智能体 (Tool Agent)
    - 分组 (Group)
    - 便签 (Note)

### 3.2 智能体节点 (Agent Nodes)
- **会话交互**：每个节点是一个独立的聊天窗口，支持实时消息发送。
- **状态管理**：支持最小化、颜色标记、加载状态显示。
- **拖拽移动**：支持自由拖拽，位置实时保存。
- **n8n 风格连线**：节点两侧设有连接点，支持通过拖拽连线建立逻辑关系。

### 3.3 组织与标注 (Organization & Annotation)
- **分组 (Groups)**：
    - 矩形区域，可容纳多个节点。
    - 支持调整大小、重命名、颜色自定义。
    - 移动分组时，内部节点联动（规划中）。
- **便签 (Notes)**：
    - 快速记录想法或任务说明。
    - 支持多种背景颜色切换。

### 3.4 导航与全局工具 (Navigation & Global Tools)
- **画布雷达 (MiniMap)**：右下角实时预览全局布局，点击可快速跳转。
- **命令菜单 (Command Menu)**：`Ctrl/Cmd + K` 唤起，支持快速搜索和执行全局指令。
- **自动布局 (Auto-Layout)**：一键整理凌乱的节点，基于拓扑排序或层级结构。
- **系统日志 (System Logs)**：记录后台 Hook 触发、API 调用等底层事件。

## 4. 技术架构
- **前端**：React 18 + TypeScript + Tailwind CSS。
- **核心引擎 (Rust Core)**：使用 **Rust** 编写的高性能计算引擎，通过 **WebAssembly (WASM)** 集成到前端。
    - 负责：自动布局算法、大规模节点碰撞检测、Rust 源码静态分析。
- **动画/交互**：Framer Motion (motion/react) 处理拖拽和过渡。
- **后端**：Node.js (Express) + Vite Middleware。
- **数据库**：SQLite (Better-SQLite3) 存储节点位置、消息、连接关系。
- **AI 集成**：Google Gemini API (@google/genai)。

## 5. UI/UX 设计原则
- **技术感与专业性**：采用深色/浅色混合风格，使用 `JetBrains Mono` 字体增强技术氛围。
- **即时反馈**：所有操作（拖拽、发送、连线）均有平滑的动画过渡。
- **低摩擦力**：快捷键优先（Cmd+N, Cmd+L, Cmd+K），减少鼠标点击。

## 6. 未来规划 (Roadmap)
- **多智能体协作流**：实现真正的自动化连线触发，A 节点的输出作为 B 节点的输入。
- **全局画布导出**：支持导出为图片或 JSON 配置文件。
- **实时多人协作**：基于 WebSockets 实现多人同时在画布上编排。
- **插件系统**：支持接入第三方工具（Google Search, Maps, Python Interpreter）。
